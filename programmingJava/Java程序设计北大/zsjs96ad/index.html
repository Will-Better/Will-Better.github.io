1 程序的编辑、编译和运行

  • 源程序编辑， 可用任一文本编辑器
  • 程序编译， 使用JDK中的javac工具
  • 程序运行， 使用JDK中的java工具

1.1 JAVA工具包JDK

Java 编程的基本工具是JDK，安装后的目录结构说明：

  • Bin 该目录存放工具文件
  • Jre 该目录存放与java 运行环境相关的文件
  • Demo 该目录存放一些示例文件
  • Include 该目录存放与C相关的头文件
  • Lib 该目录存放程序库
  • Db 数据库相关

1.2 具体细节

  1. 程序编辑。 编辑器需要保证文件名要与public class的类名一致(区分大小写)

  2. 程序编译。 转换为字节码(bytecode)文件，扩展名.class

    • .class文件中包含 java虚拟机的指令
    • 编译可以使用JDK工具javac.exe => javac Hello.java

  3. 程序的运行。 执行 .class文件中的指令的过程。=> java Hello

    注意:不要写成 java Hello.class， 因为这里需要的是类名，不是文件名

相关信息

设定path和classpath

  • path是命令(javac及java)的路径
  • classpath是所要引用的类的路径
  • 可以在命令行上设定。 set path=.;c:\jdk\bin;...
  • 也可以在系统环境中设定 > 电脑-属性-高级-环境变量

classpath

在javac及java命令行上使用-classpath (或-cp)选项可以引用别的库 javac –cp libxx.jar 源文件名.java 使用libxx.jar库

2 Java的几个工具

2.1 工具简介

  • 主要的工具
    • javac 编译
    • java 运行(控制台及图形界面程序)
    • javaw 运行图形界面程序
  • 另外常用的几个工具
    • jar 打包工具
    • javadoc 生成文档
    • Javap 查看类信息及反汇编

2.2 jar打包

  1. 编译 javac A.java
  2. 打包 jar cvfm A.jar A.man A.class
    • c表示创建(create), v表示显示详情(verbose), f表示指定文件名, m表示清单文件
    • A.man 是清单文件(manifest),包含 Manifest-Version/Class-Path/Main-Class。清单文件可以任意命名，常见的是用 MANIFEST.MF
  3. 运行 java -jar A.jar

2.3 JavaDoc生成文档

javadoc –d 目录名 xxx.java

标注

/** */ 这其中可以用以下标记

  • @author 对类的说明 标明开发该类模块的作者
  • @version 对类的说明 标明该类模块的版本
  • @see 对类、属性、方法的说明 参考转向，也就是相关主题
  • @param 对方法的说明 对方法中某参数的说明
  • @return 对方法的说明 对方法返回值的说明
  • @exception 对方法的说明 对方法可能抛出的异常进行说明
